## Внесені зміни (актуально)
1. `index.js`
   - Додано підтримку `APP_HOST`/`HOST` для керування мережевим bind-адресом сервера.
   - Дефолт змінено на `0.0.0.0`, щоб сервіс був доступний з мережі на CentOS.
   - Лог запуску тепер показує URL у форматі `http://<SERVER_IP>:PORT` для зовнішніх інсталяцій.
2. `README.md`
   - Переписано розділи з запуском на CentOS актуальними кроками.
   - Додано пояснення, як викликати API не через `localhost`, а через `SERVER_IP:PORT`.
   - Додано приклад налаштування firewall та systemd.

## Що ще треба зробити
1. Додати `.env.example` з усіма обов'язковими змінними (`PORT`, `APP_HOST`, `BASE_URL`, `CHROME_BIN`).
2. Додати валідацію payload для endpoint-ів (`/registerwhatsapp`, `/sendmsg`) через `zod` або `joi`.
3. Додати захист API (token auth + rate limit).
4. Додати інструкцію міграції/резервного копіювання `.wwebjs_auth` і `.wwebjs_cache`.
5. Додати health endpoint з деталізованими причинами (`qr_timeout`, `auth_failure`, `browser_crash`).
